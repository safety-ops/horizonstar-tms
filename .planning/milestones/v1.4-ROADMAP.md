# Milestone v1.4: Web TMS Restyle (Stripe/Linear)

**Status:** ✅ SHIPPED 2026-03-13
**Phases:** 19-26
**Total Plans:** 28

## Overview

v1.4 restyles every page of the Web TMS to a clean, flat, Stripe/Linear aesthetic. The work begins with a token foundation swap in CSS variables (60-70% instant visual shift), then sweeps through dispatch pages (where 80% of user time is spent), fleet/people pages, finance pages, operations pages, and finally shared chrome (sidebar, modals, login). Each phase delivers a fully restyled, visually consistent slice of the application. Quality gates are verified in the final phase after all pages match the target aesthetic.

## Milestone Summary

**Key Decisions:**
- Token swap first (60-70% global visual shift), then page-by-page sweeps
- Shared chrome (sidebar, modals, login) restyled LAST to avoid Frankenapp
- Light mode only, dark mode deferred to v1.5
- CSS + JS render function changes allowed (not CSS-only like abandoned v1.3)
- Stripe/Linear aesthetic: neutral/monochrome, flat surfaces, dark slate buttons
- Per-page atomic commits, never batching multiple pages
- Tailwind Slate palette for all neutrals
- Paystub print template explicitly excluded from restyle

**Issues Resolved:**
- All 32 decorative keyframes removed (particle rain, bounce, shimmer, driveTruck, etc.)
- Duplicate CSS blocks deleted (TOPBAR ENHANCEMENTS, ENHANCED NAVIGATION, Cinematic Premium modal)
- Sidebar converted from dark (#0a1014) to light (var(--bg-secondary)) theme
- Login particle animation system fully removed (CSS + JS)
- Modal CSS consolidated from 3+ definitions to single authoritative block

**Technical Debt Incurred:**
- renderFinancialAnalysis (~1200 lines) missed by Phase 24 sweep -- still has 18 gradients, hardcoded colors
- renderFixedCosts uses legacy stat-card and bare table
- 144x font-weight:700 + 39x font-weight:800 in JS template literals
- theme-driver-parity.css overrides 10 Phase 19 tokens (split source of truth)
- 205x --dim-* naming inconsistency (vs Phase 19 --*-dim convention)
- Triple badge definition (base.css dead code, two inline blocks)
