<task type="auto">
  <name>Task 1: Code-level quality verification sweep</name>
  <files>index.html</files>
  <action>
Perform a systematic code-level sweep for regressions and inconsistencies. This is a READ-HEAVY task — only modify code if actual issues are found.

**1. Badge color verification:**
Search for all badge class usages (badge-green, badge-amber, badge-red, badge-blue, badge-gray, badge-purple) in index.html. Verify they reference CSS classes defined in base.css that use CSS variables. Confirm no inline style overrides break badge colors.

Count instances: badge-green (~44), badge-amber (~45), badge-red (~36), badge-blue (~37), badge-gray (~51), badge-purple (~13). If counts deviate significantly from these baselines, investigate what changed.

**2. Remaining dark-theme holdovers:**
Search index.html for these patterns that indicate missed dark-theme CSS:
- `background: #0a1014` or `background:#0a1014` (dark background)
- `background: #030808` (login dark bg)
- `color: white` in CSS rules (not in JS template strings where white-on-colored-badge is intentional)
- `rgba(255,255,255,0.` in CSS rules (dark-theme opacity patterns)
- `linear-gradient` in CSS rules (gradients should be removed except data viz)

For each finding, determine if it's:
a) In a CSS rule (should be fixed) -> fix it
b) In a JS template string for a badge/pill on a colored background (intentional) -> leave it
c) In a tooltip or dropdown that intentionally needs dark background (intentional) -> leave it

**3. Font weight audit:**
Search for `font-weight: 700`, `font-weight: 800`, `font-weight: 900`, `font-weight:700`, `font-weight:800`, `font-weight:900` in CSS style blocks (not in JS). Cap any found at 600. Skip any that are inside `@font-face` declarations.

**4. Border radius consistency:**
Search for `border-radius: 20px`, `border-radius: 24px`, `border-radius:20px`, `border-radius:24px` in CSS rules. These should be reduced to 12px or 16px max (cards use 12px, pills use 999px). Exception: mobile bottom-sheet modal (16px 16px 0 0 is correct).

**5. Shadow consistency:**
Search for `box-shadow` in CSS rules. Verify they use `var(--shadow-xs)`, `var(--shadow-sm)`, or `var(--shadow-md)` tokens. Fix any that use heavy custom shadows (e.g., rgba with >0.3 opacity, multiple layers with glow effects).

Fix any issues found. If no issues found, do not make changes.

**Do NOT** change any JavaScript logic, data fetching, or event handlers. Only fix CSS properties and HTML class attributes.

  <done>Code-level sweep complete. All dark-theme holdovers fixed. Font weights capped at 600. Border radii normalized. Shadows use tokens.</done>
